types

F. types

Source:
  • functional.js, line 270

This namespace provides a list of identity functions for common Data Types. You may use the available utilities to validate the argument(s) of a function.

Examples

Argument validation - Integer Data Type

// The expected data type of a and b is an Integer
function add(a, b) {
   return sjs.F.types.int(a) + sjs.F.types.int(b);
}

Argument validation - Array Data Type

// The expected data type is of the a argument is an Array
function myFunc(a) {
   return sjs.F.types.arr(a).reduce((prev, curr) => prev + curr, 0);
}

Source:

Provides the User s way to specify a list of allowed Data Types for a specific argument.

Example
function checkDate(value) {
  const types = sjs.allowedTypes('String', 'Date');
  // value must be a String or a Date
  const date = new Date(types(value));
  return date;
}

Allows the User to define a new identity function for a custom Data Type

Example

  Array = sjs.types.arr(Array);
  Observer = mutObserver(Observer);
